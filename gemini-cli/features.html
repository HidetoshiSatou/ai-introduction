主な機能

code

ソフトウェア開発

バグ修正、機能追加、リファクタリング、テスト作成などを、プロジェクトの規約に合わせて行います。

例：複雑な条件分岐のリファクタリング
// Before
if (user.role === 'admin' || user.role === 'editor') {
    // ... do something
}

// After
const canEdit = ['admin', 'editor'].includes(user.role);
if (canEdit) {
    // ... do something
}
analytics

コードベース分析

専門ツールを使い、複雑なコードベースの構造理解、アーキテクチャの可視化、依存関係の解析を行います。

例：プロジェクトのエントリーポイントを調査
# ユーザーからの指示
"このプロジェクトのエントリーポイントはどこ？どうやって起動するの？"

# 私が実行するツール
codebase_investigator(
  objective="Find the entry point of the application."
)
description

ファイル操作

ファイルの読み書き、検索、そして正確なテキスト置換まで、様々なファイル操作を実行します。

例：設定ファイル内のバージョン更新
# 指示
"package.jsonのreactのバージョンを18.2.0に更新して"

# 私が実行するツール
replace(
  file_path='package.json',
  old_string='"react": "18.1.0"',
  new_string='"react": "18.2.0"'
)
terminal

コマンド実行

テストの実行、プロジェクトのビルド、あるいはその他の必要なタスクのために、シェルコマンドを実行します。

例：プロジェクトのテストを実行
# 指示
"ユニットテストを実行して結果を教えて"

# 私が実行するツール
run_shell_command(
  command='npm test'
)
travel_explore

Webリサーチ

Web検索を行い、情報収集、ドキュメントの検索、または問題の解決策を調査します。

例：ライブラリの公式ドキュメントを検索
# 指示
"Reactの公式サイトで`useEffect`の使い方を調べて"

# 私が実行するツール
google_web_search(
  query='React useEffect official docs'
)
checklist

タスク管理

複雑なリクエストを計画に分解し、TODOリストで進捗を管理します。

例：新規プロジェクトのセットアップ
# 指示
"Reactで新しいWebアプリを作って"

# 私が作成するTodoリスト
1. ViteでReactプロジェクトを初期化
2. UIコンポーネントを設計・実装
3. API通信サービスを実装
4. ...
